# ytmusic-playlist-archive [![Workflow status](https://github.com/vitokorn/ytmusic-playlist-archive/actions/workflows/main.yml/badge.svg)](https://github.com/vitokorn/ytmusic-playlist-archive/actions/workflows/main.yml)

> Daily snapshots of public YouTube Music playlists

YouTube Music playlists are useful precisely because they change: tracks get
added, removed, reordered, and refreshed over time. That is great while you are
listening today, but not so great when you want to find the song that was there
last week.

This repository keeps plain-text, cumulative, and readable snapshots of public
YouTube Music playlists so playlist history stays visible.

Special thanks to
[mackorone/spotify-playlist-archive](https://github.com/mackorone/spotify-playlist-archive),
the Spotify playlist archive project this repository was originally based on.

## Quick Start

1. To view the current version of a playlist, click on its name below.
1. To see all songs that ever belonged to a playlist, click `cumulative`.
1. To determine which songs were added or removed from a playlist, click `githistory`.
1. To add a playlist to the archive, create an empty file named after the playlist ID in `playlists/plain/`.

For example, a YouTube Music playlist URL like:

```text
https://music.youtube.com/playlist?list=PLexamplePlaylistId
```

would be archived by creating:

```text
playlists/plain/PLexamplePlaylistId
```

Radio/watch playlists are also supported when their playlist IDs start with
`RD`, such as `RDAMVM...`.

## How It Works

The script scrapes YouTube Music playlists with
[`ytmusicapi`](https://github.com/sigma67/ytmusicapi) and writes the results
back to the repository. It can be run locally, by GitHub Actions, or by any
scheduled job that has access to the repository.

The script determines which playlists to scrape by looking at file names in
`playlists/plain`. Invalid playlist IDs are skipped. Valid playlist IDs are
regenerated into three formats:

- `playlists/pretty`: readable Markdown snapshots of the current playlist.
- `playlists/cumulative`: every track that has ever appeared in the playlist.
- `playlists/plain`: stable plain-text output sorted alphabetically.

The plain output only changes when tracks are added or removed, which makes
[Git History](https://githistory.xyz/) a convenient way to inspect playlist
changes over time.

## Development

This project uses Python 3.14 and keeps dependency pins in both `uv.lock` and
`requirements/`.

Create and activate a virtual environment:

```sh
python3.14 -m venv venv
source venv/bin/activate
```

Install the development requirements:

```sh
pip install -r requirements/requirements-dev.txt
```

To refresh dependency pins:

```sh
uv lock --upgrade
uv pip compile requirements/requirements.in --python-version 3.14 --upgrade --output-file requirements/requirements.txt
uv pip compile requirements/requirements-dev.in --python-version 3.14 --upgrade --output-file requirements/requirements-dev.txt
```

To update playlist snapshots locally:

```sh
python script.py
```
